#312026-04-05T21:38:51.710Z

Are these models ever able to be run in the browser? Or is this something that generally happens on a server, or a decent machine rather than in browser?

TTS models can run in the browser, but with real trade-offs:

What works in-browser today: - Piper — there's a WebAssembly port (piper-wasm) that runs entirely client-side. Quality is identical, but inference is 3–5x slower than native since WASM doesn't fully utilize SIMD. On a modern laptop it's usable; on mobile it's rough. - Kokoro — runs via ONNX Runtime Web (WebAssembly + WebGPU backend). With WebGPU on a decent GPU-equipped machine it's actually fast. Without GPU it's slow but functional. - Small VITS models — similar story, ONNX Runtime Web handles them fine.

What doesn't work well in-browser: - Anything over ~200MB — model download before first use is a non-starter for casual users - Models requiring custom CUDA kernels or complex pre/post-processing - Mobile devices — even medium models are painful

The practical split:

Scenario Best approach
Personal tool / power users Server (what you have now)
Public demo, desktop users Either — WASM/WebGPU is viable
Public demo, mobile users Server — no question
Offline / privacy-first app In-browser WASM

For your use case specifically — since this is a public eval tool on a shared CPU server, server-side is the right call. The main risk in-browser would be that a phone user waits 30 seconds for a 150MB model to download and then another 30s to generate. Not a great first impression.

If you wanted to go in-browser for a future version, Kokoro via ONNX Runtime Web + WebGPU would be the one worth trying — it's the best quality-to-size ratio and the ONNX ecosystem has solid browser support.

#322026-04-05T21:40:28.560Z

Does anyone have a tool like this that you know of?

A few exist but none quite like what you've built:

Closest things: - Hugging Face Spaces — individual model demos exist (e.g. for Kokoro, Parler-TTS), but they're one model at a time, no parameter sliders, no side-by-side comparison, no rating system. You have to bounce between tabs to compare. - TTS Arena (by Hugging Face) — blind A/B listening test to rank models, but it's fixed samples, no parameter control, and you're voting on quality not tuning settings. - Coqui Studio (now defunct) — had a decent UI but was commercial, closed source, and is gone. - elevenlabs.io / play.ht — commercial, closed models, no parameter exposure.

What you have that none of them do: - Sliders for the actual synthesis parameters, not just voice selection - Rating system tied to specific parameter combos - Comparison matrix of your personal top picks - History with exact settings so you can reproduce anything - Self-hosted, so you control the models and compute

The gap in the market is specifically the parameter exploration angle — most tools treat TTS as a black box and just let you pick a voice. The fact that yours exposes noise_scale, noise_w, and length_scale with instant feedback is genuinely useful for anyone trying to understand what these knobs actually do, which is the stated goal.

If you open sourced it with a good README it'd likely get traction in the self-hosting / local AI communities (r/selfhosted, Hacker News, Hugging Face etc).
